# ideaLab AI Skill

这不是一个简单的素材包，而是一套给同事、顾问和 Agent 使用的 ideaLab 产品知识与赛事表达系统。

它解决五个问题：

1. 怎么讲清 ideaLab 的产品体系，而不是只罗列课程名称。
2. 怎么把青创赛、雏鹰杯、宋庆龄少年儿童发明奖讲成 Lab 赛事包的“三赛联动”能力模型。
3. 怎么引用官方数据，同时避开保奖、夸大获奖率、混淆合同边界等高风险表达。
4. 怎么读取城市/学校校情，把小升初、综评、科创信号和产品规划连接起来。
5. 怎么训练顾问：模拟家长、考核产品介绍能力、给回答打分，并示范更好的表达。

## 一句话定位

ideaLab 不是单个课程，而是一条从兴趣启蒙、原创项目、深度科研到赛事服务的科创成长路径。Lab 赛事包不是押单场比赛，而是把青创赛、雏鹰杯、宋庆龄少年儿童发明奖放进同一个项目成长矩阵，让学生的选题、研究、材料、答辩和临场表达变成可管理的交付系统。

```text
使用 ideaLab。请先检查 https://github.com/Amasun93/ideaLab 是否有比本地更新的版本；如果本地不是最新版，请提醒我更新后再回答最新价格、课时、合同、退费和赛事节点。
```

### 2. 顾问训练

适合让 Agent 扮演家长，训练顾问的产品介绍、异议处理和风险边界。

常用触发词：

- "开始顾问考核。"

- "我回答完以后给我打分。"
- "按 100 分制评价我的产品介绍。"

### 3. 模拟与示范系统

适合让顾问反过来扮演家长，看一版标准顾问回答。

常用触发词：

- "我来当家长，你示范顾问怎么答。"
- "给我一版面向家长的标准回答。"
- "这句话怎么说更合规？"
- "刚才我的回答哪里有风险？"
- "给我下一轮家长追问。"
- "把这段话改成顾问可以直接说的版本。"

### 4. 轻量治理与口径校准

适合在钉钉群或本地对话中处理“这个回答不对”“小芮，校准口径”“这里要更新”的情况。

常用触发词：

- "小芮，校准口径：……"
- "这个回答不对，正确应该是……"
- "小芮，更新知识库：……"
- "这个价格/退费/合同口径要核验一下。"

处理原则：普通咨询只回复；错别字、别名、格式和表达优化可直接更新；价格、课时、合同、退费、升学、赛事当年节点等高风险内容先进入待核验，不直接覆盖正式口径。

## 给同事的表达逻辑

讲 ideaLab 产品时，先不要从价格和课时开始。推荐顺序是：

1. 先判断孩子处在哪个阶段：兴趣启蒙、短期作品、原创发明、深度科研、赛事冲刺。
2. 再判断家长的真实目标：培养能力、做出作品、准备综评、冲赛事、要申报服务。
3. 再选择产品线：智造万物 / MYO / 科创启航 / 科创领航 / 高中课题营 / 寒(暑)假竞赛营 / 专项营 / 科创发明营。
4. 最后再讲交付边界：是否包含申报、查新、报告、答辩、退费承诺，以及哪些不能承诺。

这套顺序很重要。直接从“某个产品多少钱”开始，容易把长期能力培养和赛事服务混在一起，也容易把退费承诺讲成获奖承诺。

## 顾问训练系统怎么用

这个 Skill 不只是给顾问查资料，也可以当成一套训练场。

### 1. 知识查询

顾问可以直接问：

> 青创赛的官方数据有哪些？
> 三年级孩子适合启航计划还是竞赛营？
> 宋庆龄少年儿童发明奖怎么介绍给家长？

Agent 会先判断问题属于产品、赛事、推荐还是销售边界，再读取对应知识库。

### 2. 顾问考核

顾问可以说：

> 开始顾问考核，模拟一个家长来问我。

Agent 会从场景队列里扮演不同类型家长，例如低龄启蒙家长、价格敏感家长、想冲青创赛的家长、追问保奖的家长。也可以从家长常问问题库里抽题，例如“ideaLab到底是什么”“三项赛事是不是都要参加”“青创赛对综评有没有用”“不获奖退费是不是保奖”。顾问回答后，Agent 按 100 分制打分，维度包括产品匹配、知识准确、推荐逻辑、风险边界和沟通表达。

### 3. 示范回答

顾问也可以反过来练：

> 我来当家长，你示范顾问怎么答。

这时 Agent 会先给一版面向家长的回答，然后拆解这版回答背后的判断逻辑：为什么这样开口、为什么推荐这个产品、哪些内容不能说。

### 4. 复盘训练

每次考核后，Agent 不只给分，还会给：

- 最关键的问题。
- 更好的顾问回答示范。
- 下一轮家长追问。

这样同事不是背一段话术，而是在反复训练“听懂家长目标、匹配产品、守住边界”的能力。

- 这些资料是内部规划资料，不是学校官方招生简章。

## Lab 赛事包怎么讲

Lab 赛事包的核心不是“某个比赛一定拿奖”，而是三项赛事考察的能力不同，组合后能覆盖一个学生项目从研究到表达的多个面向：

| 赛事 | 内容表达重点 | 可以支撑的产品叙事 |
|---|---|---|
| 青创赛 | 上海市级科创赛事，重视项目研究能力、原创性、问辩和综合素养。 | 说明学生项目要经受真实申报、初审、问辩和现场表达。 |
| 雏鹰杯 | 少先队系统下的科创展示和小院士评选，强调层层筛选、研究过程和表达。 | 说明项目不只是做出来，还要让孩子能讲清楚自己怎么想、怎么研究。 |
| 宋庆龄少年儿童发明奖 | 全国性青少年科技活动奖项，覆盖发明、AI编程、创意作品和终评答辩。 | 说明项目可以进入更高密度的全国性发明创造语境。 |

推荐表达：

> Lab赛事包不是押单场，而是通过青创赛、雏鹰杯、宋庆龄少年儿童发明奖三赛联动，把学生上岸机会变成一套可管理、可交付的系统。

## 不能说的话

- 不说“保奖”“保证获奖”“100%拿奖”。
- 不把“退费承诺”说成“结果保证”。
- 不把 `92%+` 说成官方赛事综合获奖率。
- 不公开解释内部计算细节，例如剔除最低值、具体老师表现、复杂均值算法。
- 不用三项赛事不同分母直接计算“官方三赛任一获奖概率”。
- 不发布学生照片、证书、姓名、学校等隐私素材，除非已经拿到授权。

把下面这段话直接发给同事，让他们复制给自己的 AI 助手：


```powershell
git clone https://github.com/Amasun93/ideaLab "$env:USERPROFILE\.codex\skills\ideaLab"
```

只要目录下有 `SKILL.md`，Agent 下次启动就会自动加载这个 Skill。安装后，在新对话里让 Agent 使用 `ideaLab`。

## 仓库结构

```text
ideaLab/
  SKILL.md                 # 核心文件：元数据 + Agent 指令
  skill.json               # 机器可读索引：能力、关键词、引用入口
  agents/openai.yaml       # Codex/Agent UI 元数据
  references/
    product_knowledge/     # ideaLab 产品结构化知识库
    event_knowledge/       # Lab 赛事包与三项赛事资料库
    school_knowledge/      # 校情蓝皮书城市卡、学校资料卡和结构化索引
    consultant_training/   # 顾问模拟、常问问题、考核、评分、钉钉 SOP 索引
  README.md
```

## 维护规则

更新产品报价、合同、退费、赛事通知或获奖数据时，不要只改 README。应同步修改：

- `references/product_knowledge/data/*.json`
- `references/product_knowledge/产品卡片/*.md`
- `references/event_knowledge/data/*.json`
- `references/event_knowledge/90_长期素材预备/*.md`
- `references/school_knowledge/**/*.md`
- `references/school_knowledge/data/*.json`
- `references/consultant_training/*.json`
- `references/consultant_training/dingtalk_internal_docs.md`

README 只负责帮助人理解表达逻辑；真正给 Agent 检索和引用的，是 `references` 里的结构化知识库。
