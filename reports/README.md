# 小芮运营周报体系

## 目录结构

```
reports/
├── README.md                    ← 入口索引（本文件）
├── 分类体系_v1.md               ← 问题分类标准（P/TP/TA/C/$/O）
├── daily/                       ← 每日记录（周报原始数据）
│   └── YYYY-MM-DD.md
└── weekly/                      ← 周报输出
    ├── 周报_2026W27.html        ← HTML 仪表盘（Tailwind + Chart.js）
    └── 周报_2026W27.md          ← Markdown 存档

> ⚠️ 周报文件命名格式：`周报_YYYYWNN.扩展名`（如 周报_2026W27.md）
```

## 周报生成流程

收到「出周报」「本周数据」「生成周报」等请求时：

1. `git pull` 确保数据最新
2. 读取 `分类体系_v1.md` 获取分类标准
3. 读取 `weekly/` 下最新的 `.html` 文件作为设计模板
4. 读取本周所有 `daily/` 日记录
5. 按模板格式生成新周报（HTML + MD），写入 `weekly/` 目录
   - 命名格式：`周报_YYYYWNN.html` 和 `周报_YYYYWNN.md`
6. git add + commit + push 推送到 GitHub
7. 🚨 自动将 HTML 周报文件发送到「游击队」群（cidRHm0F13522JAlQ27w9YftQ==）

## 分类体系

| 代码 | 分类 | 说明 |
|------|------|------|
| P | 产品咨询 | 产品定位、内容、课时 |
| TP | 赛事规划 | 为具体学生推荐赛事路径 |
| TA | 赛事分析 | 数据驱动的区域/学校对比 |
| C | 学情参谋 | 学校分析、学区规划 |
| $ | 价格课时 | 价格、合同、退费 |
| O | 教务行政 | 排课、占座、报名、发货 |

## HTML 模板核心模块

1. 概览卡片（总提问、活跃顾问数、覆盖群数、新用户数）
2. 分区域活跃度（进度条 + 顾问名单）
3. 用户活跃度 TOP3
4. 问题分类饼图（Chart.js）
5. 高频问题 TOP 表格
6. 区域趋势简析（三色卡片）
7. 本周学生案例追踪
8. 下周建议（培训+产品+内容）
