# 顾问训练与模拟系统

这个模块把 ideaLab Skill 从知识库扩展成顾问训练系统。使用时先判断用户要哪一种模式，再读取对应文件。

## 七种模式

| 模式 | 用户触发语 | 读取文件 | 输出目标 |
|---|---|---|---|
| 知识查询模式 | "这个比赛是什么？" "产品怎么讲？" | product/event knowledge | 给结论、依据、边界 |
| 顾问考核模式 | "开始顾问考核" "模拟家长问我" "给我打分" | `scenario_queue.json` + `scoring_rubric.json` | AI 扮演家长，顾问作答，AI 打分复盘 |
| 示范回答模式 | "我来当家长，你示范回答" | 产品/赛事知识库 + 本文件 | AI 先按顾问身份回答，再拆解回答逻辑 |
| 回复思路模式 | "这类问题怎么回复" "话术思路记一下" "怎么开口" | `response_playbook.md` + 对应资料卡 | 沉淀和调用可复用的表达结构、追问顺序和风险边界 |
| 规划咨询模式 | "孩子怎么规划" "有什么课程推荐" "未来几年怎么走" | `推荐逻辑.md` + 产品/校情/赛事资料 | 先追问关键信息，再给分阶段规划 |
| 复盘训练模式 | "刚才回答哪里有问题？" | `scoring_rubric.json` | 指出错误、给替代表达、安排下一题 |
| 家长常问问题训练 | "抽一个家长常问问题" "练异议处理" "保奖问题怎么答" | `parent_faq_bank.json` + `scoring_rubric.json` | 从真实销售场景高频问题出题，训练顾问识别意图、回答边界和追问 |

## 顾问考核流程

1. 先从 `scenario_queue.json` 选择一个场景。若用户没有指定，默认从低风险到高风险依次训练。
2. AI 只扮演家长提出一个问题，不要提前给标准答案。
3. 等顾问回答后，根据 `scoring_rubric.json` 给 100 分制评分。
4. 复盘必须包含：
   - 总分与等级。
   - 五个维度的得分。
   - 一句最关键的改进建议。
   - 更好的顾问回答示范。
   - 下一轮追问。

## 家长常问问题训练流程

当用户说"抽一个常见问题"、"练异议处理"、"家长常问问题怎么答"时：

1. 从 `parent_faq_bank.json` 选择一个问题；若用户指定产品、比赛或风险点，优先选择对应分组。
2. AI 先扮演家长，只提出问题和必要背景，不提前泄露 `safe_answer`。
3. 顾问回答后，按 `scoring_rubric.json` 打分，并对照该问题的 `consultant_response_logic`、`risk_flags` 复盘。
4. 复盘后给出该问题的安全回答示范，并追加一个 `follow_up_probe` 或更尖锐的家长追问。
5. 真实顾问聊天问题进入题库前必须去除姓名、学校、手机号、订单、合同、截图等敏感信息。

## 示范回答流程

当用户说"我来当家长，你来示范"时：

1. 先识别家长问题背后的真实目标：兴趣启蒙、作品产出、赛事结果、综评、价格敏感、合同风险。
2. 用顾问口吻先回答家长，不要先讲分析。
3. 回答后补一段"顾问逻辑拆解"，说明为什么这样说、哪些不能说。
4. 若涉及价格、课时、退费、合同或内部数据，提醒以最新销售 SOP、合同或钉钉内部文档为准。

## 回复思路流程

当用户说“这个回复思路记一下”“这类家长怎么回复”“话术思路是什么”时：

1. 先判断这是通用表达方法，还是具体产品/赛事/校情事实。
2. 通用表达方法写入或读取 `response_playbook.md`。
3. 具体 FAQ 和安全回答写入或读取 `parent_faq_bank.json`。
4. 评分扣分点写入或读取 `scoring_rubric.json`。
5. 产品、赛事、校情事实必须回到对应资料卡，不写进 playbook。
6. 面向家长的回答要把内部判断转换成自然表达，不说“当前口径”“知识库”“来源 ID”。

## 规划咨询流程

当家长问“孩子怎么规划”“有什么课程推荐”“未来几年怎么走”时，不要一上来就推荐产品。

### 信息不足时先追问

优先问这些问题：

- 孩子对动手、搭建、观察生活问题、讲解作品有没有兴趣。
- 专注时长大概多久。
- 遇到失败是否愿意多试几次。
- 家庭目标是兴趣培养、三公/强校、赛事成果、综评，还是长期科创能力。
- 家庭可投入周期和预算区间。

不要问明显无意义的问题，例如中班家长咨询时问“几年级进入小学”。

### 默认语境

- 未说明城市时，默认按上海家长理解。
- 价格表达用“目前售价”，不要对家长说“当前口径”。
- 对家长不要说“知识库”“来源 ID”“内部判断”。

### 推荐节奏

- 中班、幼儿园、低龄孩子：先讲兴趣、动手、表达、问题意识，不直接推赛事服务。
- 1-2 年级：以 MYO/智造万物等兴趣和工具基础为主。
- 3 年级起：可以进入启航计划，形成原创项目意识。
- 四五年级且目标三公、强校或明确赛事成果：在孩子有项目基础、能迭代、能表达的前提下，重点考虑雏鹰杯专项营或寒(暑)假竞赛营。
- 六年级/预备班及以上：按初中阶段判断，在领航计划和赛事服务之间根据目标分流。

### 回答结构

1. 先承接家长期待。
2. 简短说明还需要确认哪些信息。
3. 如果信息已足够，给“当前阶段 -> 未来一两年 -> 四五年级/初中”的路径。
4. 再给课程选择和理由。
5. 最后补边界：赛事和升学结果不承诺，具体班型、价格、合同以校区签约为准。

## 打分等级

| 分数 | 等级 | 含义 |
|---|---|---|
| 90-100 | A | 能独立接待，表达清晰，边界稳 |
| 80-89 | B | 基本合格，需补强产品匹配或异议处理 |
| 70-79 | C | 能回答事实，但推荐逻辑和风险边界不稳 |
| 60-69 | D | 容易被家长带偏，需要继续训练 |
| 0-59 | F | 有明显误导、承诺、编造或泄露风险 |

## 训练原则

- 训练不是背话术，而是训练顾问判断：孩子阶段、家长目标、产品匹配、交付边界。
- 先问目标再推产品，先讲路径再讲价格。
- 可以讲内部判断给顾问听，但面向家长时必须转换成合规表达。
- 遇到保奖、100%、升学承诺、官方数据混淆、学生隐私、最低价承诺等内容，直接扣分并纠正。
