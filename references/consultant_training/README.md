# 顾问训练与模拟系统

这个模块把 ideaLab Skill 从知识库扩展成顾问训练系统。使用时先判断用户要哪一种模式，再读取对应文件。

## 四种模式

| 模式 | 用户触发语 | 读取文件 | 输出目标 |
|---|---|---|---|
| 知识查询模式 | "这个比赛是什么？" "产品怎么讲？" | product/event knowledge | 给结论、依据、边界 |
| 顾问考核模式 | "开始顾问考核" "模拟家长问我" "给我打分" | `scenario_queue.json` + `scoring_rubric.json` | AI 扮演家长，顾问作答，AI 打分复盘 |
| 示范回答模式 | "我来当家长，你示范回答" | 产品/赛事知识库 + 本文件 | AI 先按顾问身份回答，再拆解回答逻辑 |
| 复盘训练模式 | "刚才回答哪里有问题？" | `scoring_rubric.json` | 指出错误、给替代表达、安排下一题 |

## 顾问考核流程

1. 先从 `scenario_queue.json` 选择一个场景。若用户没有指定，默认从低风险到高风险依次训练。
2. AI 只扮演家长提出一个问题，不要提前给标准答案。
3. 等顾问回答后，根据 `scoring_rubric.json` 给 100 分制评分。
4. 复盘必须包含：
   - 总分与等级。
   - 五个维度的得分。
   - 一句最关键的改进建议。
   - 更好的顾问回答示范。
   - 下一轮追问。

## 示范回答流程

当用户说"我来当家长，你来示范"时：

1. 先识别家长问题背后的真实目标：兴趣启蒙、作品产出、赛事结果、综评、价格敏感、合同风险。
2. 用顾问口吻先回答家长，不要先讲分析。
3. 回答后补一段"顾问逻辑拆解"，说明为什么这样说、哪些不能说。
4. 若涉及价格、课时、退费、合同或内部数据，提醒以最新销售 SOP、合同或钉钉内部文档为准。

## 打分等级

| 分数 | 等级 | 含义 |
|---|---|---|
| 90-100 | A | 能独立接待，表达清晰，边界稳 |
| 80-89 | B | 基本合格，需补强产品匹配或异议处理 |
| 70-79 | C | 能回答事实，但推荐逻辑和风险边界不稳 |
| 60-69 | D | 容易被家长带偏，需要继续训练 |
| 0-59 | F | 有明显误导、承诺、编造或泄露风险 |

## 训练原则

- 训练不是背话术，而是训练顾问判断：孩子阶段、家长目标、产品匹配、交付边界。
- 先问目标再推产品，先讲路径再讲价格。
- 可以讲内部判断给顾问听，但面向家长时必须转换成合规表达。
- 遇到保奖、100%、升学承诺、官方数据混淆、学生隐私、最低价承诺等内容，直接扣分并纠正。
