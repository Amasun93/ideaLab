# 赛事资料库 README

更新时间：2026-06-26

这个资料库服务于内容生产，不只是保存原始素材。Agent 读取时应优先理解三层结构：

1. 事实层：每个赛事的官方/准官方来源、赛事定位、评审流程、可引用数据。
2. 关系层：青创赛、雏鹰杯、宋庆龄少年儿童发明奖共同构成 Lab 赛事包的“三赛联动”叙事。
3. 表达层：哪些句子可用于口播、图文、社群答疑，哪些内部计算或敏感比较不能对外写。

## Agent 快速使用

- 先读 `SKILL.md`，再读 `三赛联动_Lab赛事包/02_Lab赛事包叙事卡.md`。
- 写某个赛事时，进入对应赛事文件夹读 `00_赛事简介.md` 和 `01_官方数据卡.md`。
- 需要机器读取时，使用 `data/events.json`、`data/data_points.json`、`data/claim_bank.json`。
- 需要核对本地新增的学情通知、赛事通知、获奖/入围名单批次时，使用 `data/local_event_sources_20260626.json` 和 `data/local_event_award_stats_20260626.json`；这两份只提供脱敏索引和聚合统计，不提供名单明细。
- 需要视觉素材时，先读 `data/visual_manifest.json`，不要把未标为 official_logo 的图片当作官方 logo。

|赛事|赛事定位|内容生产用途|
|---|---|---|

## Lab 赛事包核心口径

|范围|可用表达|风险|改写边界|
|---|---|---|---|
|Lab赛事包|Lab赛事包不是押单场，而是把青创赛、雏鹰杯、宋庆龄少年儿童发明奖放进同一个项目成长矩阵。|低|可以讲三赛联动；不要讲成官方联合认证或单个赛事保奖。|
|Lab赛事包|Lab赛事包通过三赛联动，把团队综合上岸率稳定做到九成以上；数据展示可写“92%+”。|中|不要写去掉最低值、哪个老师被剔除、复杂计算过程；对外发布需补统计说明。|
|公开赛事对比|公开赛事里，单项自然获奖率常常只有三成到四成；有些全国性奖项更低。Lab做的是把机会变成可管理的系统。|中|不要用三项赛事不同分母直接计算官方综合概率；不要暗示Lab 92%由官方单项概率推导出来。|

## 目录说明

- `青创赛/`：上海市青少年科技创新大赛资料、数据卡、规则入口、准官方新闻图。
- `雏鹰杯/`：“雏鹰杯”红领巾科创达人挑战赛与上海少年科学院小院士资料。
- `宋庆龄少年儿童发明奖/`：官方章程、奖项设置、第二十届数据、官网视觉素材。
- `三赛联动_Lab赛事包/`：三赛关系、Lab 赛事包口径、颁奖视频可用句、禁用表达。
- `data/`：给 Agent 和脚本读取的结构化数据。
- `data/local_event_sources_20260626.json`：2026-06-26 本地赛事通知与获奖资料批次的脱敏索引，只记录赛事类别、资料类型、文件类型、隐私标记和使用边界。
- `data/local_event_award_stats_20260626.json`：上述批次中 Excel 名单类资料的聚合奖项词频和候选记录量估算，仅作内部检索和趋势判断辅助。
- `source_snapshots/`：关键网页快照。网页版权归原站点所有，本地仅作内部核验。

## 重要限制

- 公开来源不能直接计算“一个学生三赛任一获奖”的官方综合概率，因为三项赛事分母不同、参赛人群不同、奖项层级不同。
- `92%+` 是内部产品结果口径，不是公开赛事官方数据。对外发布前需要补统计期、分母和隐私审查。
- 青创赛已补官方平台主视觉和从主视觉裁切出的标题字样；独立官方 logo 原文件仍未确认，不使用来路不明图片冒充官方 logo。
- 2026-06-26 本地批次包含大量名单、学情、学校、指导老师和可能的学生身份信息。GitHub 发布包不包含原始文件、原文件名、行样例、证书、学生姓名、学校、指导老师或学籍号；若要追溯明细，只能在本地私有清洗库和原始归档中处理。
- `local_event_award_stats_20260626.json` 中的奖项词频是机器辅助统计，不是官方获奖总数。对外引用数字时，仍要优先使用 `data_points.json` 和 `source_registry.json` 中已核验的公开来源。
