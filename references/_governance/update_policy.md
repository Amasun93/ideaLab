# ideaLab 知识库更新规则

更新时间：2026-07-02

本文件定义“什么时候回答、什么时候入库、什么时候待确认”。当前维护人是用户和 Serry/信芮，不启用复杂管理员审核。

## 基本原则

1. 普通顾问提问：只回答，不改文件。
2. 用户、Serry/信芮或其他同事明确要求“更新、校准、入库、清洗、沉淀”时：进入维护流程。
3. 不能识别发言人身份时：按消息意图、风险等级和来源可信度处理。
4. “更新到 skill”默认指更新 ideaLab 知识包，不等于直接改 `SKILL.md`。
5. `SKILL.md` 只改路由、模式和硬边界；事实内容写入 `references/`。

## 三档处理

| 档位 | 触发场景 | 动作 |
|---|---|---|
| A 只回复 | 正常问产品、规划、赛事、校情、赛情、话术 | 基于资料库回答，不改文件 |
| B 可更新 | 错别字、别名、格式、表达优化、公开来源、已确认低风险事实 | 更新对应资料卡或数据文件 |
| C 待确认 | 高风险字段、来源不足、口径冲突、隐私或法律风险 | 写入 `pending_updates.md`，不覆盖正式口径 |

## 低风险内容

可以直接更新：

- “小瑞”改为“小芮”。
- 错别字、标点、格式。
- 低风险别名，例如“宋赛=宋庆龄少年儿童发明奖”。
- 不改变事实的表达优化。
- 官方公开链接加入来源库。
- 已废弃旧称补充到 aliases。
- 资料卡字段补齐。

## 高风险内容

以下内容不得仅凭一句群聊反馈直接覆盖正式口径：

- 价格。
- 课时。
- 合同。
- 退费。
- 保奖、获奖率、升学、综评、自招、录取。
- 当年赛事截止时间、报名入口、资格条件。
- 校区具体执行政策、排班、占座、库存。
- 学生、学校、老师、证书、学籍号、手机号、订单、合同等隐私。
- 竞品负面事实、法律风险表达。

处理方式：

1. 记录到 `pending_updates.md`。
2. 标明待确认问题、涉及文件、建议动作和所需来源。
3. 用户或 Serry/信芮明确确认后，再更新正式资料卡。

## 建议校准格式

同事希望小芮更新时，建议提供：

```text
小芮，校准口径：
【问题场景】
【错误回答】
【正确口径】
【适用范围】顾问答疑 / 家长咨询 / 内部规划 / 内容生产
【是否可外发】可外发 / 仅内部 / 待确认
【来源】SOP / 官方通知 / 用户确认 / Serry确认 / 公开链接
```

如果同事没有按格式写，agent 也要尽量从内容中提取这些字段。

## 写入位置

| 内容 | 写入位置 |
|---|---|
| 产品事实、课程、价格、合同、退费 | `references/product_knowledge/` |
| 赛事事实、官方数据、赛事话术 | `references/event_knowledge/` |
| 区级通知、申报时间、区域竞争 | `references/event_knowledge/区域赛事赛情/` |
| 城市/学校/班型/学校索引 | `references/school_knowledge/` |
| 顾问训练、模拟家长、评分规则 | `references/consultant_training/` |
| 竞品资料 | `references/competitive_intelligence/` |
| 历史规划案例 | `references/planning_cases/` |
| 字段、维护规则、风险规则 | `WIKI.md` 或 `references/_governance/` |
| 路由新增或硬边界变化 | `SKILL.md` |
| 关键词、版本、发布信息 | `skill.json` |

## 回答前自检

回答前判断：

1. 是否命中已有资料卡？
2. 是否涉及高风险字段？
3. 是否存在多个冲突口径？
4. 是否是当年时效性强的信息？
5. 是否需要区分公开口径和内部判断？

答案不稳时，降级表达：说明当前已知口径、来源边界和待核验点，不要编造。

## 回答后纠错处理

如果用户纠正小芮：

1. 表达问题：可更新话术。
2. 低风险事实错误：可更新并记录来源。
3. 高风险事实变化：进入待确认。
4. 与正式资料冲突：优先正式资料，用户反馈记入待确认。
5. 没有来源的断言：不进入正式资料卡。

## GitHub 同步

- 用户说“更新一下、校准一下、入库、清洗、整理、同步资料卡”等明确维护指令时，完成后默认提交并推送到 GitHub。
- 用户说“先别推、先本地讨论、先看方案、不要更新 GitHub”时，只做本地整理或方案讨论，不提交推送。
- 推送前检查没有原始未脱敏材料、临时清洗文件或隐私数据。
